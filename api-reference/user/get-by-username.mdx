---
title: "Получить пользователя по username"
api: "GET http://207.154.237.52:8000/user/get-user-by-username/{username}"
description: "Получение данных пользователя по его username"
---

## Параметры пути

<ParamField path="username" type="string" required>
  Имя пользователя (username)
</ParamField>

## Ответ

<Note>
  В отличие от `/user/get-user/{id}`, этот эндпоинт **не возвращает** поля `password` и `status`.
</Note>

<ResponseField name="id" type="string">
  Уникальный идентификатор пользователя (UUID)
</ResponseField>

<ResponseField name="username" type="string">
  Имя пользователя
</ResponseField>

<ResponseField name="email" type="string">
  Email пользователя
</ResponseField>

<ResponseField name="nickname" type="string">
  Никнейм пользователя
</ResponseField>

<ResponseField name="user_avatar" type="string">
  Путь к аватару пользователя
</ResponseField>

<ResponseField name="user_banner" type="string">
  Путь к баннеру пользователя
</ResponseField>

<RequestExample>
```bash cURL
curl -X 'GET' \
  'http://207.154.237.52:8000/user/get-user-by-username/whyroma' \
  -H 'accept: application/json'
```

```javascript JavaScript
const response = await fetch(
  "http://207.154.237.52:8000/user/get-user-by-username/whyroma",
  {
    headers: { Accept: "application/json" },
  }
);
const user = await response.json();
```

```python Python
import requests

response = requests.get(
    "http://207.154.237.52:8000/user/get-user-by-username/whyroma",
    headers={"Accept": "application/json"}
)
user = response.json()
```
</RequestExample>
